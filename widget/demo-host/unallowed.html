✗ UNALLOWED ORIGIN — widget is blocked
Demo tip: Open DevTools → Console. You should see a CSP violation error. The iframe is created by the loader script, but the browser refuses to load its content because http://localhost:8081 is not in frame-ancestors.

This page is served from http://localhost:8081, which is not listed in the widget's allowed_origins. Even though the loader script runs and creates the <iframe>, the browser enforces the Content-Security-Policy: frame-ancestors header returned by GET /widget/config/{id} and blocks the widget from rendering.

This prevents malicious sites from embedding your widget and hijacking visitor interactions.
